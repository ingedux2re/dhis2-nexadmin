## How to Add a New Language

**7 steps — takes about 2 hours for a complete translation:**

1. **Copy the source file**
   ```bash
   cp i18n/en.po i18n/es.po   # replace 'es' with your language code
   ```

2. **Translate all `msgstr` values**
   Open `i18n/es.po` in any text editor or PO editor (Poedit recommended).
   For each entry, translate the `msgstr` line:
   ```po
   msgid "Health Dashboard"
   msgstr "Panel de salud"   # ← your translation here
   ```

3. **Register the locale** in `src/constants/i18n.ts`:
   ```typescript
   { code: 'es', label: 'Español', flag: '🇪🇸', pluralForms: 'nplurals=2; plural=(n != 1)' }
   ```

4. **Regenerate the JS bundle**
   ```bash
   yarn d2-app-scripts i18n generate
   ```

5. **The LanguageSwitcher** will automatically show the new button — no code change needed.

6. **Test** by switching to the new language in the app:
   ```bash
   yarn start
   # Click the new language button in the top bar
   ```

7. **Submit PR**:
   ```bash
   git add i18n/es.po src/constants/i18n.ts i18n/i18n.js
   git commit -m "i18n(es): add Spanish translation"
   ```

## Workflow for Developers

After adding or changing any user-facing string in the app:

```bash
# 1. Extract new strings from source code
yarn d2-app-scripts i18n extract

# 2. Check i18n/fr.po for any new empty msgstr entries
grep -A1 "msgstr" i18n/fr.po | grep 'msgstr ""'

# 3. Translate the new entries in fr.po (and any other languages)
# 4. Regenerate JS bundle
yarn d2-app-scripts i18n generate

# 5. Include updated .po and i18n.js in your PR
```

## Interpolation (variables in strings)

Use double-brace syntax — compatible with i18next:

```javascript
// In your component:
i18n.t("{{count}} duplicates found", { count: 8 })
// → "8 duplicates found" (EN)
// → "8 doublons trouvés" (FR)
```

In the .po file:
```po
msgid "{{count}} duplicates found"
msgstr "{{count}} doublons trouvés"
```

## Plural Forms

```po
msgid "{{count}} unread notification"
msgid_plural "{{count}} unread notifications"
msgstr[0] "{{count}} notification non lue"
msgstr[1] "{{count}} notifications non lues"
```
